THE ONE-LINE IDEA
-----------------
Chain-of-thought faithfulness = whether the stated reasoning reflects the model's actual computation. Cast
the emitted CoT as a SELECTED sample of the latent computation: the model may commit to an answer latently
and then verbalize a selected/rationalized projection, not the cause. This has the structure of selection-
on-unobservables (Heckman 1979): you only observe the reasoning through the verbalized channel, which is
correlated with -- but not equal to -- the true computation, so naive faithfulness metrics (counts of
whether a factor is verbalized) are CONFOUNDED by selection into what-gets-verbalized. The contribution is
the ESTIMATOR: an explicit selection equation for what-gets-verbalized + an outcome equation for the
underlying computation, linked by correlated errors, yielding (a) a TEST for the confound (H0: rho=0) and
(b) an inverse-Mills-ratio-style CORRECTION to naive faithfulness metrics, validated against activation-
patching ground truth on open 8B models and then applied to observation-only API models.

THE DISTINGUISHING STATEMENT (put a version in the intro)
--------------------------------------------------------
"The field has established that a model's chain-of-thought is a selected, often post-hoc projection of its
latent computation, and separately that faithfulness metrics are confounded by the lossy selectivity of
verbalization. We give the first formal account of this as ENDOGENOUS SAMPLE SELECTION: a selection equation
for what-gets-verbalized and an outcome equation for the underlying computational reliance, linked by
correlated errors -- yielding a scalar TEST for the verbalization-selection confound (H0: rho=0) and an
inverse-Mills-ratio CORRECTION to naive faithfulness estimates, validated against activation-patching ground
truth on open models and applied to observation-only API models."

EXPERIMENT LADDER (gated; E0 is a WEEKEND go/no-go)
---------------------------------------------------
E0. WEEKEND GO/NO-GO (the confound exists + the correction helps). On Llama-3.1-8B (4-bit, fits the 3080)
    over MMLU / ARC-Easy/Challenge / OpenbookQA with Turpin-style HINT INJECTION (2305.04388):
    - For each instance record: V = verbalized-hint INDICATOR (the selection OUTCOME -- did the CoT mention
      the hint/factor it actually used?).
    - GROUND-TRUTH latent reliance (the "unobservable made observable" anchor): a cheap causal proxy on the
      open model -- logit-lens "answer flips to the hint before the hint is verbalized" and/or a single
      activation-PATCH of the hint token (NDE/NIE-style effect size, a la 2512.23032). This is the outcome the
      verbalization is a selected projection OF.
    - Fit the two-equation selection model (probit selection eq for V on instance features [hint type,
      difficulty, position] + outcome eq for reliance) with an EXCLUSION RESTRICTION (see below); TEST H0:
      rho=0. THE PAPER EXISTS IF: naive verbalization-based faithfulness rates are BIASED, and the IMR-
      corrected estimate (and/or the rho test) recovers the patch-based ground truth BETTER than the raw
      verbalization count. If rho~0 everywhere / the correction does not beat the raw count -> K1, no confound
      to correct, project weak.
E1. IDENTIFICATION HARDENING. The exclusion restriction is the load-bearing assumption: a manipulation that
    shifts WHAT GETS VERBALIZED but NOT the latent computation. Candidates: decoding-budget / instruction-to-
    be-concise / temperature. Validate the exclusion (show the manipulation moves V but not the patch-based
    reliance). Per econometrics best practice, ALSO report rho-SENSITIVITY (results across fixed rho values)
    rather than lean on joint-normality identification. Pre-register this as the chief limitation.
E2. OBSERVATION-ONLY TRANSFER (the point of the correction). Apply the correction WITHOUT patching access on
    Nemotron (Llama-Nemotron reasoning) + Claude via API -- the ideal "selection-only-observed" regime the
    method is FOR: you see only the verbalized channel and must correct the faithfulness estimate. Show the
    correction changes the faithfulness verdict in a direction consistent with the open-model ground truth.
E3. ROBUSTNESS: >=3 models (Llama-3.1-8B, Qwen-2.5-7B, a Gemma), multiple hint types (from 2505.05410's six),
    bootstrap CIs, and a placebo (a "hint" that carries no information -> the correction should find no
    confound). Machine-generated numeric macros.

PRE-REGISTERED KILL CONDITIONS (commit to PLAN.md BEFORE results; commit that file BEFORE the results commit)
-----------------------------------------------------------------------------------------------------------
K1. NO CONFOUND: rho is indistinguishable from 0 across setups AND the IMR-corrected faithfulness does not
    beat the raw verbalization count at recovering patch-based ground truth -> there is no selection confound
    to correct; honest negative (the naive metric was fine), still a small methods result.
K2. EXCLUSION-RESTRICTION FAILURE: the proposed instrument (budget/temperature) demonstrably ALSO changes the
    latent computation (moves the patch-based reliance) -> identification fails; fall back to a rho-sensitivity
    "bounds" result rather than a point-identified correction, and say so.
K3. REDUCES-TO-MNAR-MEDIATION: if under scrutiny the estimator is numerically identical to an existing
    missing-mediator correction with no added test/closed-form value -> reframe honestly as "the selection-
    correction packaging of MNAR mediation for CoT" (still novel-as-application) or demote.
